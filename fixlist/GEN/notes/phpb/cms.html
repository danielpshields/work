Content Management Systems (CMS) vs. Traditional Code

The better option is determined by the level of control the owner needs over the Site once it's completed. Either option will require some work:

CLAS WordPress Site

The CLAS WordPress theme will need to have code written to format the Site to support its original contents. It needs a repopulating of appropriately formatted media. It will also require restoring custom features like the secondary navigation display, drop down behaviors and interactive graphics.

Pros

  • Features / Plugins: simple access to advanced features like forms
  • Web Accessibility Standards: immediate / instant
  • Users: anyone can add, delete and edit pages

Cons

  • simple tasks are more complex | i.e. WYSIWYG -- Text editing is relatively simple, but formatting efficacy, like found in a word processor, can be difficult to obtain/implement
  • complex / custom tasks are more complex | i.e. SVGs -- Advanced features can require more work due to having to work within a CMS environment
  • CMS problems become PHPB Site problems

AAC Traditional Code Site

The AAC traditional code version will need to score higher on Web accessibility standards. And it will potentially need be restyled to mimic the current CLAS theme's settings.

Pros

  • structure: established, malleable / dynamic Site structure
  • custom anything: Custom features are easier to build and implement

Cons

  • Theme: the color scheme differs from the CLAS WordPress theme and uniformity is a strength in a departmental Website
  • Accessibility: the web accessibility need to be updated to score higher on standards
